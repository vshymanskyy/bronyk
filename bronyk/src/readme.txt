Вирішив виписати сюди загальні правила, певний досвід роботи з С++, чого не можна і як треба.
От списочок:

 1. Глобальні ф-ції і дані це в 98% погано. У випадку глобальних об'єктів краще використовувати Singleton.

 2. Використовувати по максимуму слово const (як модифікатор даних так і методів)

 3! При оголошенні абстрактних інтерфейсів обовязково писати пустий вірт. деструктор (див ctrl/abstr/*.h).

 4. Не забувати звільняти виділені ресурси (зокрема, память).

 5. Якшо реалізація ф-ції в хедері обовязково писати її з модифікатором inline або static.

 6. В кожному хедері (*.h) першою лінійкою має бути хочаб "#pragma once" (щоб хедер)

 7. Кому не ліньки, бажано розібратися з процесом зборки програми
      Тобто що таке "сторінка компіляції" і процес
      препроцесор -> компілятор -> лінкер
      (сюди вставити ссилку на якусь норм статтю)

 8. При виділенні ресурсів старатися використовувати конструкцію:
	if (SomeRes = AllocSomeRes()) {
		...
		ReleaseSomeRes();
	}
      Наприклад:
	if (char* str = new char[1024]) {
		...
		delete[] str;
	}

 9! Зберігати відповідність операторів виділення та чистки памяті:
      Якщо память виділена оператором new то чистити тільки через delete
      Якщо память виділена оператором new[] то чистити тільки через delete[]
      Якщо память виділена malloc(), calloc(), strdup(), ... (Сішними ф-ціями) то чистити тільки через free()

10! Практично _ніколи_ не можна викликати "delete this;", а також викликати вручну конструктори і деструктори.

11. Треба фіксити warning-и при компіляції, часом вони не такі вже й невинні :)

12! Не можна повертати з ф-ції референс на локальні дані. (Компілятор навіть видає warning)
      ТАК НЕ РОБИТИ:
	Obj& SomeFunc(int param) {
		Obj result;
		...
		return result;
	}

13. Бажано в конструкторах використовувати списки ініціалізації.
      Наприклад:
	class A {
		int a;
		int b;
	public:
		A (int valB)
			: a (1)
			, b (valB)
		{}
	};
